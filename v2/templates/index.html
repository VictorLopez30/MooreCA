Guía rápida del sistema

Entropía de Shannon

Mide qué tan uniforme o impredecible es la distribución de valores de la imagen cifrada.

En imágenes RGB de 8 bits, un valor cercano a 8 bits indica que los valores aparecen con frecuencia parecida.

Chi-cuadrada

Compara el histograma observado contra una distribución ideal uniforme.

Valores más bajos suelen indicar menos sesgo en las intensidades; valores altos sugieren concentración en ciertos tonos.

Correlación

Evalúa la relación entre píxeles vecinos. En una imagen natural suele ser alta porque los píxeles cercanos se parecen.

En una imagen cifrada se busca que esté cerca de 0, indicando baja dependencia espacial.

Histograma RGB

Muestra cuántas veces aparece cada valor de intensidad de 0 a 255 en los canales rojo, verde y azul.

Sirve para detectar picos, saturaciones o tendencias visibles en la imagen cifrada.

Volver a métricas
Cómo funciona el cifrado

1. Imagen de entrada

La imagen se convierte a datos RGB crudos. El cifrado trabaja sobre los bytes de píxeles, no sobre el archivo comprimido directamente.

2. Llaves de sesión

Se obtiene un secreto base de 32 bytes y un salt de 32 bytes. Con HKDF-SHA256 se derivan llaves para permutación, autómata y sesión.

La interfaz permite usar una sesión independiente por lenguaje o una sesión compartida para los tres.

3. Permutación

En cada ronda se elige una de 128 permutaciones. Esta reorganiza espacialmente los píxeles mediante desplazamientos por filas y columnas.

4. Autómata de Moore

Después de permutar, se aplica una generación del autómata celular con vecindad de Moore y aritmética módulo 257.

Cada ronda usa coeficientes derivados de la llave, por lo que la evolución cambia ronda a ronda.

5. Rondas

Una ronda equivale a una permutación más una generación del autómata. Aumentar rondas incrementa la difusión, pero también el tiempo de cómputo.

6. Descifrado

El sistema guarda dos estados finales y un archivo de sesión. Con esos datos se revierte el autómata y se recupera la imagen original.

Las métricas ayudan a observar comportamiento estadístico, pero no sustituyen un análisis criptográfico formal. En esta interfaz se usan como apoyo para comparar difusión, uniformidad y recuperación entre las implementaciones en C, Java y C#.
